# Tier 2: generic (fallback)

Use when nothing fits and a custom archetype would be overkill.

**Stack:** whatever the user specifies
**Key files:** depends on tech
**Skills:** minimum — `/devlog`, `/new-system`, `/obsidian-canvas`
**Wiki folders:** base — Systems/, Architecture/, Devlog/, Canvas/
**Triggers:** only universal ones (wiki maintenance, focus tracker)
**Pitfalls (universal):**
- No README
- No .env.example
- No LICENSE
- Outdated deps
- TODOs without a date
- Comments missing "why" where logic is complex
